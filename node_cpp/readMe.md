## 研究c++程序和node程序的通信方式

### nodejs为主进程 c++为子进程

| 方案                              | 速度          | 复杂度 | 适用场景         |
| :-------------------------------- | :------------ | :----- | :--------------- |
| **stdin/stdout + JSON**           | 慢 (~10MB/s)  | 低     | 控制命令、配置   |
| **Unix Domain Socket + Protobuf** | 快 (~500MB/s) | 中     | 一般数据传输     |
| **Shared Memory + 信号量**        | 极快 (~GB/s)  | 高     | 大数据、实时视频 |
| **Memory Mapped File**            | 快            | 中     | 大文件、持久化   |

### Node.js C++ Addon vs 独立子进程

| 特性           | C++ Addon (node-gyp)    | 独立 C++ 子进程            |
| :------------- | :---------------------- | :------------------------- |
| **进程关系**   | 同进程，共享 V8 堆内存  | 独立进程，完全隔离         |
| **崩溃影响**   | 会导致 Node.js 崩溃     | Node.js 完全不受影响       |
| **内存访问**   | 直接访问 V8 对象        | 必须通过 IPC/共享内存      |
| **适用场景**   | 高频小数据调用          | 大数据处理、不稳定代码隔离 |
| **构建复杂度** | 高（需要适配 Node-API） | 低（标准 C++ 即可）        |

### 核心对比：Addon vs 独立进程

```plain
┌─────────────────────────────────────────────────────────────┐
│              Node.js C++ Addon                              │
├─────────────────────────────────────────────────────────────┤
│  Node.js 进程                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  V8 引擎                                             │   │
│  │  ┌─────────────┐    ┌──────────────────────────┐   │   │
│  │  │  JS 代码    │◄──►│  C++ Addon (.node 文件)   │   │   │
│  │  │             │    │  - 直接调用 CUDA          │   │   │
│  │  │             │    │  - 阻塞主线程 ❌          │   │   │
│  │  │             │    │  - 崩溃 = 整个 Node 崩溃  │   │   │
│  │  └─────────────┘    └──────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                  │
│                    问题: 阻塞事件循环                       │
│                    问题: 难以利用多核                       │
│                    问题: CUDA 错误导致整个应用崩溃           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│              共享内存 + 独立 C++ 进程  ✅                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Node.js 主进程              C++ 计算进程 (独立)           │
│   ┌───────────────┐           ┌───────────────────────┐    │
│   │  事件循环      │◄────────►│  独立进程              │    │
│   │  永不阻塞      │  Unix     │  - 专用 CUDA GPU      │    │
│   │  崩溃隔离      │  Socket   │  - 多线程并行计算      │    │
│   │               │           │  - 崩溃可重启          │    │
│   └───────────────┘           └───────────────────────┘    │
│          │                              │                   │
│          └────────► 共享内存 ◄──────────┘                   │
│                (零拷贝大数据传输)                            │
│                                                             │
│   优势: 完全隔离、充分利用 GPU、Node 永不崩溃               │
└─────────────────────────────────────────────────────────────┘
```

### 完整实现：共享内存 + Unix Socket

#### 架构确认

```plain
Node.js 主进程                    C++ 计算进程
     │                                  │
     │──── Unix Socket (命令通道) ─────►│
     │   "process_buffer", offset=0     │
     │                                  │
     │◄──────── 结果返回 ───────────────│
     │    {status: "ok", time: 12.3}    │
     │                                  │
     └────────┐    ┌───────────────────┘
              ▼    ▼
     ┌─────────────────────┐
     │  /dev/shm/shared.dat │  ← 共享内存 (mmap)
     │  [大块数据: 图像/张量] │
     │  零拷贝，双方直接读写  │
     └─────────────────────┘
```
